1. User can create wallets, or delete or edit them.
2. User can view their wallets.
3. User can see the budgets related to the wallet if the wallet is clicked. If the wallet is not clicked, then all wallets are shown.
4. When the user clicks on a wallet, the budgets of that wallet are shown, along with the top ten transactions of the selected wallet.
5. Each wallet can have multiple budgets.
6. Enable transfer of money from one wallet to another.
7. User can see the top ten transactions of the selected wallet.
8. If no wallets are found, then show the create wallet button.
9. **Wallet Details**: Users can view detailed information about each wallet, including transaction history, budget allocations, and balance over time.
10. **Search and Filter**: Users can search for specific wallets or filter them based on criteria such as balance, type, or creation date.
11. **Notifications**: Users receive notifications for significant events, such as low balance alerts or successful transactions.
12. **Error Handling**: If the user has no wallets, display a message encouraging them to create a new wallet.
